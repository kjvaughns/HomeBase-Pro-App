HomeBase Homeowner Tools Upgrade: Survival Kit + HouseFax Ledger + Savings Insights
Design only. React Native Expo. iOS-first. Liquid Glass. Calm luxury minimalism. No white outlines. No loud glows. Soft green accents only for primary actions and positive states. Use mock data and fully clickable flows.

Objective
Design the Survival Kit and HouseFax Ledger system so homeowners can:
1. Add and maintain a rich Home Profile per address
2. Track total spend on home services over time
3. See estimated savings from using HomeBase
4. Store home history tied to the specific address and use it to predict future maintenance and costs
5. Get an estimated yearly maintenance cost based on home details, neighborhood context, and house type, plus tips to reduce that cost

Global rules
1. Everything is per Home address. User can have multiple homes and switch homes from a persistent Home selector.
2. All screens are design only but must be interactive with mock data and realistic empty states.
3. Keep design consistent with the current Homeowner portal. Liquid Glass surfaces, soft shadows, clean typography, restrained motion.
4. Every output must connect to one tap booking with a service category and home_id (visual only). Buttons: Book Now, Get Quote, Add to Plan, Snooze, Mark Done.
5. Store “address history” conceptually: timeline of updates, jobs, invoices, inspections, assessments, and changes to home fields. Design the UI for it.

Navigation placement
Add a “Tools” section inside the Homeowner More tab with four tiles:
1. Survival Kit
2. HouseFax Ledger
3. Savings and Spend
4. Home Health Score (placeholder tile if not building yet)

Also add quick entry tiles on Home screen under “Your Home” for:
Survival Kit and HouseFax Ledger

Tool 1: Survival Kit
Purpose
A guided wizard that gathers home details and outputs:
1. Estimated yearly maintenance cost range
2. A personalized maintenance plan with tasks
3. Tips to reduce costs and prevent emergencies
4. Cost drivers explanation so it feels trustworthy

Entry Screen
Card header: “Survival Kit”
Subtext: “Build your maintenance plan and predict yearly upkeep costs.”
Primary CTA: Start
Secondary: Use my saved home details

Wizard Steps (design a 12 to 18 step wizard, fast but meaningful)
Step groups with progress indicator and micro animations:
A. Home basics
Address (select existing home or add new)
Property type (single family, townhome, condo, duplex)
Year built or home age
Square footage
Bedrooms, bathrooms
Stories
B. Key systems
HVAC type and age
Water heater type and age
Roof age and material
Plumbing type optional
Electrical panel type optional
Appliances optional
C. Exterior and risks
Yard size
Trees near roof yes/no
Basement yes/no
Pool/spa yes/no
Garage yes/no
D. Neighborhood context (design only, mock derived fields)
Neighborhood type selector: urban, suburban, rural
Climate zone selector: hot, cold, mixed
Local risk chips: hail, flooding, humidity, freezing, wildfire (multi select)
E. Preferences
DIY comfort level (low, medium, high)
Service level: lean, standard, proactive
Budget goal: keep costs low, balanced, protect resale

UX rules for wizard
Single question per screen with large tap targets
Use saved Home profile values as defaults
Explain why each input matters in one short line
Add 3 instant gratification moments:
“You are ahead of 82% of homeowners”
“Your plan will prevent the top 3 costly surprises”
“HomeBase will auto remind you and make booking one tap”
