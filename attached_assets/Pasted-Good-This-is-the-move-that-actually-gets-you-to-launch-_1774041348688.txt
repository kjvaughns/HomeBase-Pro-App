## Phase 1: Core Flow Validation

Provider signs up → completes onboarding → creates service → sets availability → shares booking link → homeowner books → provider manages job → payment is completed

* confusing UX points
* missing backend logic
* unreliable states

## Phase 2: Frontend Audit

Review all screens and flows:

Identify:

* broken UI components
* inconsistent design
* bad navigation
* confusing layouts
* slow or laggy screens
* mobile responsiveness issues

## Phase 3: Backend + Infrastructure Audit

Analyze backend systems including Supabase, Stripe, APIs, and overall architecture.

Check:

### Auth

* secure login and signup
* session handling
* role separation (provider vs homeowner)

### Database

* clean schema structure
* proper relationships (users, services, bookings, payments)
* no redundant or unused tables
* scalability risks

### Booking System

* booking creation logic
* availability conflicts
* edge cases (double booking, timezone issues)

### Payments (Stripe)

* payment flow works end to end
* provider payouts (Stripe Connect if used)
* webhook handling
* failed payment handling
* refund logic (basic)

### APIs

* error handling
* rate limits
* timeouts
* fallback states

## Phase 4: Critical Edge Cases

Identify and test:

* double bookings
* user cancels last minute
* provider no-shows
* failed payments
* partial payments / deposits
* timezone mismatches
* user switching roles
* incomplete onboarding accounts

List what breaks and how to fix it.

## Phase 5: Performance + Stability

* app load time
* screen transition speed
* API response times
