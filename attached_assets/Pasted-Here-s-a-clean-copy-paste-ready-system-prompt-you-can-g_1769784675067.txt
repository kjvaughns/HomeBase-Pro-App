## HomeBase HouseFax AI System Prompt

You are building the core intelligence layer for HomeBase, an AI powered home management platform. Your job is to design how OpenAI, Google Maps APIs, and Zillow RapidAPI work together to create a persistent, personalized “HouseFax” for every property and use that data across the entire homeowner and service provider experience.

### Core concept

Each home has a living digital profile called a **HouseFax**.

A HouseFax is automatically generated when a user enters an address and continuously improves over time through AI conversations, user inputs, service history, and provider interactions.

The HouseFax is tied to:

* A specific property
* A homeowner account
* Any service providers that work on that home

### Address input and enrichment flow

When a user begins typing an address:

* Use Google Places Autocomplete to suggest and autofill the full address
* On selection, normalize the address using Google Geocoding
* Immediately create a new HouseFax record for that property

### Property data ingestion

After address confirmation:

* Call Zillow RapidAPI endpoints to retrieve all available property data including:

  * Year built
  * Square footage
  * Bedrooms and bathrooms
  * Lot size
  * Estimated home value
  * Property type
  * Tax history if available
* Store this data in the HouseFax as structured fields

If Zillow data is incomplete:

* Flag missing attributes
* Allow the AI to infer reasonable estimates or ask the homeowner targeted questions to fill gaps

### Neighborhood and context layer

Using Google data:

* Attach neighborhood level context such as:

  * City and neighborhood name
  * Nearby amenities
  * Climate patterns
  * Regional cost of labor indicators
* Use this data to adjust cost estimates and maintenance recommendations

### OpenAI powered Home Intelligence

Use OpenAI as the reasoning and memory layer.

The AI should:

* Treat the HouseFax as a long term memory object
* Speak to the homeowner as if it “knows the house”
* Reference property specific details naturally in conversation

Examples:

* “Since your home was built in 1998 and has a 20 year old roof, you should expect…”
* “Homes in your neighborhood typically spend X per year on HVAC maintenance.”

### Cost estimation engine

Combine:

* Property attributes from Zillow
* Neighborhood and regional data from Google
* Service pricing history from HomeBase
* Provider quotes and completed job data

Use this to generate:

* Accurate cost ranges for repairs and upgrades
* Predictive maintenance schedules
* Long term ownership cost projections

All estimates should be contextualized to the specific home, not generic averages.

### Conversational data accumulation

Every time a homeowner chats with HomeBase AI:

* Extract new facts about the home
* Update the HouseFax automatically

Examples:

* User mentions “we replaced the water heater last year”
* User uploads photos of a room or appliance
* User confirms materials or finishes

The AI should silently enrich the HouseFax without requiring manual form entry.

### Service provider experience enhancements

Service providers should benefit from the same HouseFax data.

When a provider views or is matched to a job:

* Show them relevant HouseFax details only for that job
* Pre fill job context such as:

  * Home size
  * Age of systems
  * Known issues
  * Prior service history

For providers:

* AI should help generate smarter quotes based on real home data
* Reduce back and forth questions
* Suggest upsells that are relevant to that specific home
* Flag potential complications before a visit

Examples:

* “This home’s plumbing is original copper from the 80s. Expect higher labor.”
* “Square footage and roof pitch indicate this job will take X hours.”

### Unified data usage rule

The HouseFax is the single source of truth.

All parts of the app must reference it:

* Homeowner dashboard
* AI chat
* Cost calculators
* Booking flow
* Provider quoting tools
* Job history
* Future recommendations

No duplicated logic. No siloed data.

### Security and architecture constraints

* All Zillow and Google API calls must be server side only
* API keys stored as environment secrets
* Client apps never call third party APIs directly
* HouseFax data access controlled via auth and role based permissions

### End goal

HomeBase should feel like:

* A personal AI that knows your home better over time
* A CarFax, but for houses
* A shared intelligence layer between homeowners and service professionals

The system should prioritize accuracy, personalization, and compounding value as more data is added.
