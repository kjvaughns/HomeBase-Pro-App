## DESIGN PROMPT

### HomeBase Provider Services & Public Business Profile System

**Design only. React Native Expo. iOS-first. Liquid Glass. Calm luxury.**

### Objective

Design the **Services Management system and Public Business Profile** inside the **HomeBase Provider app**, ensuring it perfectly aligns with the **Homeowner marketplace and booking experience**.

This system must feel **simple, powerful, and intelligent**, not busy or over-styled.
Avoid white outlines, harsh borders, excessive glow, or visual noise.

Design for **clarity, trust, and confidence**.

---

## Design Language & Principles

* **Dark, minimal, premium**
* Frosted Liquid Glass surfaces with subtle depth
* Soft green accent used sparingly for action and status
* No hard outlines or neon borders
* Floating elements only when they serve purpose
* Smooth, restrained animations
* Clear hierarchy over decoration

---

## SECTION 1: Services Hub (Provider View)

### Services List Screen

Purpose: Quickly understand, manage, and publish services.

**Layout**

* Header: “Services”
* Search bar with soft glass background
* Category filter chips (Cleaning, HVAC, Handyman, Plumbing, etc.)
* Floating “Add Service” FAB (single, centered, subtle glow)

**Service Cards**
Each card includes:

* Service name
* Category label
* Pricing type badge:

  * Fixed
  * Variable
  * Service Call
  * Quote Only
* Estimated price or “From $X” or “Requires Estimate”
* Published toggle (soft switch)
* Subtle arrow for edit

Card style:

* Slight glass blur
* Rounded edges
* No outlines
* Hover / press elevation only

## SECTION 2: Create / Edit Service Flow

This is a **guided, intelligent builder**, not a long form.

### Step 1: Service Basics

* Service name
* Category selector
* Short client-facing description
* Publish toggle

Optional AI helper:

* “Generate description with AI”
* Preview tone: professional, friendly, premium

---

### Step 2: Pricing Model Selection

Displayed as large selectable cards:

1. **Fixed Price**

   * Flat rate
   * Duration
2. **Variable Pricing**

   * Based on inputs like:

     * Bedrooms
     * Bathrooms
     * Square footage
     * Units
   * Dynamic estimated range preview
3. **Service Call**

   * Diagnostic fee
   * Optional hourly rate
4. **Quote Only**

   * No pricing shown
   * Requires approval

Each option shows:

* Simple explanation
* Example use case
* Clean icon
* Single active selection

---

### Step 3: Booking Rules & Intake

* Instant booking toggle
* Approval required toggle
* Intake questions builder:

  * Text
  * Yes/No
  * Number
  * Photo upload
* Require photos toggle

UX:

* Inline editable question chips
* Minimal drag handles
* No clutter

---

### Step 4: Availability & Buffers

* Duration selector
* Buffer before / after
* Recurring availability preview

---

### Step 5: Client Preview (Critical)

Show how this service looks **exactly** to homeowners:

* Service card
* Pricing behavior
* Intake questions
* Book button

This creates trust and confidence.
