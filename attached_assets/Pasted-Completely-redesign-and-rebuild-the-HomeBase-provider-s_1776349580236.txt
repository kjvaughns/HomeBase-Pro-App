Completely redesign and rebuild the HomeBase provider side Job Creator so it is fully connected to Supabase, aligned with our newly built Service Creator, and aligned with the way homeowners currently book services.

Core problem to fix:
The current Job Creator is not properly connected to the backend. It is showing services that were not created by the provider, pricing is not correctly tied to the selected service, and the flow does not feel aligned with either our Service Creator or our homeowner booking system. Right now it feels disconnected, generic, and unreliable.

What I want:
Create a provider focused job scheduling flow that uses the provider’s actual published services as the foundation, pulls real service data from Supabase, and allows the provider to schedule a job for a client in a way that feels like the provider version of the homeowner booking flow.

The provider should be able to:
1. Select an existing client or create a new client inline
2. Select one of their real services from their own service catalog only
3. Automatically inherit the service’s real pricing structure, duration, title, description, add ons, intake questions, and service metadata from Supabase
4. Adjust or override pricing if needed
5. Fill in a job specific problem description or scope summary for that particular booking
6. Add internal notes
7. Select date and time
8. Create the job and linked appointment correctly in Supabase
9. Trigger a professional confirmation email to the client with an accurate service summary, problem description, date, time, quote, and provider details

Also update the homeowner booking notification flow so that when a homeowner books a provider, they also receive a much better job summary email with the real service details, chosen add ons, intake answers, date, time, estimated price, and next steps.

Important product direction:
The provider job creator should feel like the provider side version of the homeowner booking flow, not some random internal admin form. It should be clean, structured, and powered by the provider’s existing service setup. The provider should feel assisted, not forced. It should feel simple, premium, and operationally useful.

Use this product logic:

WORKFLOW 1. HOMEOWNER BOOKS A PROVIDER
- Homeowner starts on provider public profile
- Taps Book Now
- Auth gate if needed
- Service is chosen from provider’s real published service catalog
- Add ons are selectable if applicable
- Date and time are chosen from provider availability
- Service intake questions are shown if configured on the service
- Estimated total is calculated from service pricing plus add ons
- On submit, backend creates:
  - appointment record tied to homeowner user id and home address with status pending
  - job record tied to a provider side client profile
- If the homeowner does not already exist as a client in the provider’s system, create one automatically
- Send email confirmation to homeowner with full job summary
- Send notification and email to provider about the new booking
- Create in app notification for homeowner
- Redirect homeowner to success screen

WORKFLOW 2. PROVIDER CREATES A JOB FOR A CLIENT
- Provider opens Add Job from dashboard
- Provider selects existing client or creates a new client inline
- Provider selects one of their actual services from Supabase
- The selected service should load:
  - service name
  - service category
  - pricing model
  - base price
  - add ons
  - duration
  - intake questions or custom fields if applicable
  - provider defined service description
- Provider can then enter a job specific problem description or client request summary for this booking
- Provider can optionally adjust quote or use AI price suggestion only as assistance, never as the default source of truth
- Provider selects date and time
- Provider can add internal notes
- On submit, backend creates:
  - job record
  - linked appointment record
- This should happen in one atomic transaction
- Appointment status should be confirmed because the provider is scheduling it directly
- Client should receive an email confirmation automatically with:
  - provider business name
  - service booked
  - date and time
  - quoted price
  - problem summary
  - what to expect next
- Job should appear immediately in provider schedule and client history
- Everything should be stored in Supabase and properly linked relationally
